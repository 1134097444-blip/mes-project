# 产品愿景与范围

## 1. 愿景陈述

构建一套**由 AI 驱动开发**、可私有化部署的 Steedos MES（制造执行系统），覆盖生产排程、工单执行、质量追溯、设备维保、物料流转全链路，作为离散制造企业车间级数字化统一作业平台。

## 2. 业务目标

| 编号 | 目标 | 衡量指标 | 优先级 |
|------|------|----------|--------|
| G1 | 生产过程全程可追溯 | 每件产品可回溯到工序、操作工、设备、批号 | P0 |
| G2 | 工单执行状态实时可见 | 车间看板 5 秒内反映最新报工数据 | P0 |
| G3 | 质量缺陷可追踪闭环 | 缺陷从发现到整改全流程留痕 | P0 |
| G4 | 设备维保计划自动提醒 | 到保设备提前提醒，历史记录完整 | P1 |
| G5 | 多角色权限隔离 | 操作工只见自己工单，主管见全车间 | P0 |

## 3. 范围（Scope）

### 3.1 In Scope（MVP）

- **生产执行**：工单管理、工序流转、生产报工、完工入库
- **工艺数据**：工艺路线、工序定义、BOM 物料清单
- **质量管控**：检验计划、检验记录、缺陷登记、不合格处理
- **设备管理**：设备台账、点检计划、维保记录、故障报修
- **物料管理**：物料主数据、线边库存、物料消耗
- **看板与分析**：工单看板、设备状态看板、质量统计看板
- **权限**：5 个标准角色（管理员/生产主管/操作工/质检员/设备技术员）
- **部署**：本地/私有 docker-compose

### 3.2 Out of Scope（MVP 不含）

- 高级排程（APS）集成
- 移动 App（仅响应式 Web）
- 多语言（仅 zh-CN）
- 设备 IoT 实时数据采集（仅手工记录）
- ERP 对接（接口预留，本期不实现）
- 复杂审批流
- 运行时 AI Copilot（本期仅开发期 AI）

## 4. 利益相关方

| 角色 | 关注点 |
|------|--------|
| 操作工 | 接收工单、报工、查看工艺要求、记录质量问题 |
| 生产主管 | 工单派发、进度跟踪、人员调配、产量统计 |
| 质检员 | 检验任务、缺陷登记、不合格处理、质量报告 |
| 设备技术员 | 设备点检、维保计划执行、故障维修记录 |
| 计划员 | 工单下达、物料准备、产能评估 |
| 管理员 | 权限配置、基础数据维护、系统审计 |
| 厂长 | 整体 OEE、产量、质量合格率、设备利用率 |

## 5. 假设与约束

- **假设**：车间具备基础网络环境；操作工经过基础培训；浏览器为 Chrome/Edge 最新版。
- **技术约束**：必须基于 Steedos 平台（@steedos/server），不可重写底座。
- **进度约束**：MVP 8 周内交付（4 个阶段，每阶段 2 周）。
- **资源约束**：以 AI 生成为主，人工评审，无专职前端/后端团队。
