# 用户故事与用例

格式：`作为 <角色>，我希望 <能力>，以便 <价值>`。优先级：P0 必须、P1 应该、P2 可选。

## 1. 生产执行模块

| ID | 故事 | 优先级 | 阶段 |
|----|------|--------|------|
| US-P01 | 作为生产主管，我希望创建并下达生产工单，以便车间开始生产 | P0 | Phase 1 |
| US-P02 | 作为操作工，我希望在工单中按工序报工，以便记录自己的劳动产出 | P0 | Phase 1 |
| US-P03 | 作为生产主管，我希望看到所有工单的进度状态，以便及时调度 | P0 | Phase 1 |
| US-P04 | 作为操作工，我希望在工单详情中看到工艺要求和作业指导，以便按标准操作 | P0 | Phase 2 |
| US-P05 | 作为计划员，我希望将工单关联到具体设备和工艺路线，以便生产过程规范化 | P1 | Phase 2 |
| US-P06 | 作为生产主管，我希望工单完成后自动更新设备使用记录，以便设备管理有据可查 | P1 | Phase 3 |

## 2. 工艺数据模块

| ID | 故事 | 优先级 | 阶段 |
|----|------|--------|------|
| US-R01 | 作为工艺工程师，我希望创建和管理工艺路线，以便定义产品的加工流程 | P0 | Phase 1 |
| US-R02 | 作为工艺工程师，我希望在工艺路线中按顺序定义工序，以便工单可以按路线流转 | P0 | Phase 1 |
| US-R03 | 作为工艺工程师，我希望为每道工序设定标准工时，以便估算产能 | P1 | Phase 2 |
| US-R04 | 作为工艺工程师，我希望维护 BOM（物料清单），以便工单领料有据可依 | P1 | Phase 3 |

## 3. 质量管控模块

| ID | 故事 | 优先级 | 阶段 |
|----|------|--------|------|
| US-Q01 | 作为质检员，我希望创建检验记录并关联到工单或设备，以便质量数据可追溯 | P0 | Phase 2 |
| US-Q02 | 作为质检员，我希望在检验中发现缺陷时登记详细缺陷信息，以便后续分析 | P0 | Phase 2 |
| US-Q03 | 作为质检主管，我希望对严重缺陷启动不合格处理流程，以便追溯整改 | P1 | Phase 3 |
| US-Q04 | 作为质检主管，我希望查看质量统计看板（合格率、缺陷分布），以便分析质量趋势 | P1 | Phase 4 |
