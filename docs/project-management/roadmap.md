# 项目路线图 — MES 制造执行系统

## 目标

本路线图用于管理 MES 项目的阶段性交付。所有开发任务必须以 `docs/design.md` 为设计基线，并在 `docs/project-management/tasks.md` 中登记和跟踪。

## 阶段规划

| 阶段 | 名称 | 目标 | 状态 | 关联文档 |
|------|------|------|------|----------|
| Phase 1 | 基础工程与主数据 | 建立 Steedos 基础工程、MES 包、应用导航、基础权限集、生产工单、设备台账、物料主数据 | 已完成 | `phase-1-foundation.md` |
| Phase 2 | 工艺质量与执行 | 实现工艺路线、工序定义、报工记录、检验记录、缺陷登记 | 已完成 | `phase-2-production.md` |
| Phase 3 | 深度业务闭环 | 实现 BOM 清单、不合格处理、维保记录、故障报修、线边库存 | 已完成 | `phase-3-quality-equipment.md` |
| Phase 4 | 看板分析与收尾 | 实现车间看板、质量统计看板、设备状态看板、种子数据 | 已完成 | `phase-4-analytics.md` |

## 阶段进入标准

- 前一阶段任务已全部进入"已完成"状态。
- 本阶段任务已在 `tasks.md` 中登记。
- 本阶段涉及的设计变更已更新到 `design.md`。

## 阶段完成标准

- 阶段内任务均已完成验收。
- 新增对象均以 `mes_` 开头。
- 新增对象均包含对象定义、字段、权限、列表视图和导航入口。
- 中文文档已同步更新。
- 已使用项目已有命令完成必要验证。
